# 🛡️ MSDS 검색 시스템

> **물질안전보건자료(Material Safety Data Sheet) 전문 검색 및 관리 시스템**

전문적이고 체계적인 MSDS 관리를 위한 웹 기반 검색 플랫폼입니다. 1000개 이상의 MSDS를 효율적으로 검색, 분류, 관리할 수 있도록 설계되었습니다.

## ✅ 현재 구현된 기능

### 🔍 검색 기능
- **실시간 검색**: 타이핑하는 동안 즉시 결과 표시
- **다중 필드 검색**: 제품명, 제조사, CAS 번호, 키워드로 검색
- **검색어 하이라이트**: 검색 결과 강조 표시
- **검색 초기화**: 원클릭으로 검색 초기화

### 📂 필터링 시스템
- **17개 카테고리 분류**:
  - 용접재료, 절단/연마, 윤활유/그리스, 연료(유류)
  - 시멘트류, 콘크리트혼화제, 콘크리트 응집제, 박리제
  - 품질시험, 스프레이류, 가스류, 요소수, 부동액
  - 경화제, 몰탈/접착제, 발파/폭약류, 기타

- **위험등급 필터**: 위험, 경고, 주의
- **제조사별 필터**: 모든 제조사 자동 추출
- **다중 필터 조합**: 여러 필터를 동시에 적용 가능
- **실시간 개수 표시**: 각 필터별 MSDS 개수 표시

### 📄 MSDS 카드 시스템
- **카드형 레이아웃**: 깔끔한 그리드 디스플레이
- **리스트형 레이아웃**: 상세 정보 표시 (전환 가능)
- **정보 표시**:
  - 제품명 (강조)
  - 제조사
  - 카테고리 뱃지
  - 위험등급 (색상 코딩)
  - 개정일자
  - CAS 번호

### 🔎 PDF 뷰어
- **모달 방식 뷰어**: 페이지 이동 없이 MSDS 확인
- **내장 PDF 뷰어**: 브라우저에서 바로 열람
- **상세 정보 표시**: 제품 정보 요약
- **다운로드 기능**: PDF 파일 직접 다운로드

### 📱 반응형 디자인
- **PC 최적화**: 대형 화면에서 효율적 배치
- **태블릿 지원**: 중형 화면 레이아웃
- **모바일 최적화**: 스마트폰에서도 완벽한 사용성

### 🎨 전문적인 UI/UX
- **산업 안전 테마**: 파란색/회색 계열의 전문적 디자인
- **부드러운 애니메이션**: 카드 호버 효과, 페이드인 등
- **직관적 아이콘**: Font Awesome 아이콘 사용
- **접근성**: 키보드 네비게이션, 명확한 대비

## 🚀 사용 방법

### 1. 기본 검색
1. 상단 검색바에 검색어 입력 (제품명, 제조사, CAS 번호)
2. 실시간으로 결과가 필터링됩니다
3. 검색어 옆 ❌ 버튼으로 초기화

### 2. 필터 사용
1. 카테고리, 위험등급, 제조사 중 원하는 항목 선택
2. 여러 필터를 동시에 적용 가능
3. "초기화" 버튼으로 모든 필터 해제

### 3. MSDS 보기
1. 카드의 "PDF 보기" 버튼 클릭
2. 모달 창에서 PDF 확인
3. "다운로드" 버튼으로 파일 저장

### 4. 뷰 전환
- 그리드 뷰 (📋): 카드형 레이아웃
- 리스트 뷰 (📄): 상세 정보 표시

## 📝 데이터 추가/수정 방법

### 새로운 MSDS 추가하기

**파일**: `data/msds-data.js`

```javascript
{
    id: 11, // 고유 ID (순차적으로 증가)
    productName: "제품명",
    manufacturer: "제조사명",
    category: "카테고리명", // 17개 중 하나
    hazardLevel: "위험등급", // 위험, 경고, 주의 중 하나
    casNumber: "CAS 번호", // 없으면 "-"
    revisionDate: "2024-01-15", // YYYY-MM-DD 형식
    pdfUrl: "PDF 파일 URL",
    description: "제품 설명",
    keywords: ["키워드1", "키워드2", "키워드3"] // 검색용
}
```

### 단계별 추가 방법

1. **`data/msds-data.js` 파일 열기**
2. **`msdsData` 배열의 마지막에 새로운 객체 추가**
3. **모든 필드 정확히 입력**
4. **쉼표(,) 구분 확인**
5. **저장 후 브라우저 새로고침**

### 예시: 11번째 MSDS 추가

```javascript
const msdsData = [
    // ... 기존 10개 데이터 ...
    {
        id: 11,
        productName: "SK ZIC TOP 5W-30",
        manufacturer: "SK에너지",
        category: "윤활유/그리스",
        hazardLevel: "경고",
        casNumber: "8042-47-5",
        revisionDate: "2024-03-15",
        pdfUrl: "https://example.com/zic-top.pdf",
        description: "합성 엔진오일",
        keywords: ["엔진오일", "윤활유", "5W30", "합성유"]
    }
];
```

## 🔄 1000개로 확장하는 방법

### 단계 1: 데이터 준비

#### 방법 A: 엑셀로 정리 후 변환

1. **엑셀 템플릿 생성**
   ```
   | id | productName | manufacturer | category | hazardLevel | casNumber | revisionDate | pdfUrl | description | keywords |
   ```

2. **1000개 데이터 입력**

3. **JSON 변환 도구 사용**
   - [Excel to JSON Converter](https://www.convertcsv.com/csv-to-json.htm)
   - 또는 Python 스크립트 사용

4. **변환된 JSON을 `data/msds-data.js`에 붙여넣기**

#### 방법 B: JavaScript로 직접 추가

```javascript
// 기존 10개 데이터 유지하고 계속 추가
const msdsData = [
    { id: 1, ... },
    { id: 2, ... },
    // ... 998개 더 추가
    { id: 1000, ... }
];
```

### 단계 2: PDF 파일 준비

1. **클라우드 스토리지 사용 (추천)**
   - Google Drive
   - Dropbox
   - AWS S3
   - Azure Blob Storage

2. **공개 URL 생성**
   - 각 PDF 파일의 다운로드 가능한 공개 URL 생성

3. **데이터에 URL 입력**
   ```javascript
   pdfUrl: "https://drive.google.com/file/d/xxxxx/view"
   ```

### 단계 3: 성능 최적화 (선택사항)

1000개 데이터에서도 빠른 성능을 위해:

## 📞 지원 및 문의

### 사용 중 문제가 발생하면:

1. **데이터가 표시되지 않음**
   - 브라우저 콘솔 확인 (F12)
   - `data/msds-data.js` 문법 오류 확인
   - 쉼표(,), 중괄호({}) 확인

2. **PDF가 열리지 않음**
   - PDF URL이 유효한지 확인
   - CORS 정책 확인 (클라우드 설정)
   - 브라우저 팝업 차단 해제

3. **검색이 작동하지 않음**
   - 브라우저 캐시 삭제 후 새로고침
   - JavaScript 에러 확인
